Image Filters - Public, Private and Default

Also click: Public Filters, Private Filters, Default Filter, Modifying a Filter, Custom Filter

 

Filters can be Clinical or Administrative (not both within the same filter)

  • system administrators assign users access to Clinical, Administrative (or both types of Filters)

Public filters

  • are displayed in the ‘File’ menu of the Image List window

  • are displayed to all users

  • cannot be created, modified or deleted by users

BACK

Private filters

  • saved with the User Preferences, therefore they are active on any workstation

  • found in the “Private tab” list of filters of the Image Filter Add/Edit window

  • are displayed in the ‘File’ menu of the Image List window

  • can be displayed as tabs, in the Image List window (Options menu)

  • can be created, modified and deleted

BACK

Default Filter

  • when a user ‘Saves Settings’ the current filter (unless it is a temporary ‘Custom’filter) is saved as the default filter for that user.

  • the next time the user starts VistA Imaging, their Default Filter is applied to the list of images returned for the selected patient.

BACK

Modifying a Filter

A Private Filter may be modified by selecting the filter name from the Private tab in the Image Filter Add/Edit window.

   When the user:

  •  attempts to select a different filter or

  • tries to switch to the Public tab or

  • tries to switch filter type (from either Administrative filters or Clinical filters) the following Confirm window will appear.

              

  • Yes – Saves changes and continues.

  •  No – Discards changes and continues.

  • Cancel – returns the user to the previous state -- enabling additional modification of the filter

        

A Public Filter can be selected from the Public tab in the Image Filter Add/Edit window.

  • it cannot be modified but it can, be used to build a Private filter with attributes of the Public filter.

  • the user will be required to name the newly created Private filter

      

  • upon saving the Filter, it will be placed (automatically) on the list of Private filters

 

Overwriting an existing Private filter is allowed
